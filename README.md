# fe-news

뉴스 스탠드 UI를 기반으로  
**Grid / List View 전환, 카테고리 필터링, 페이지네이션, 구독 기능**을 구현한 프론트엔드 프로젝트

---

## 프로젝트 개요

- **View**: Grid / List 탭 전환 지원
- **Data**: 앱 초기 진입 시 뉴스 데이터 1회 로드 (서버가 없는 관계로 json에서 불러옴)
- **State**: 카테고리 / 페이지 / 구독 상태 기반 렌더링
- **UI**: 롤링 탭, 구독 배지, 페이지 이동 등 인터랙션 중심

---

## 전체 구조 요약

- 뉴스 데이터는 `newsStore`에서 중앙 관리
- 각 View(Grid / List / Rolling)는 store 데이터를 참조해 렌더링
- 페이지, 카테고리, 구독 상태를 기준으로 렌더링 기준을 명확히 분리

---

## 구현 완료 기능

### 1. 기본 UI / 레이아웃
- 시멘틱 태그 기반 구조 설계
- 상단 뉴스스탠드 로고 + 시스템 날짜 표시
- 최신 뉴스 자동 롤링 탭 2개 구성
- Grid View (6 × 4) 뉴스 스탠드 구성
- Grid 셀 hover 시 구독 UI 노출
- Grid 좌 / 우 페이지 이동 버튼 배치

---

### 2. Grid / List View 전환
- Grid / List 전환 탭 구현
- View 전환 시 container 기준 완전 초기화 (`innerHTML = ''`)
- DOM 상태 의존 없이 View 단위 렌더링 구조로 분리

---

### 3. 롤링 탭 구조 개선
- 전역 클락(`rollingClock`) 기반 롤링 구조 설계
- 각 롤링 탭은 클락 tick을 구독하는 방식으로 동작
- hover 시 해당 영역만 일시 정지, 전역 클락 tick 무시
- hover 해제 시 별도 시간 계산 없이 자동 재동기화
- 두 롤링 탭이 항상 1초 간격을 유지하도록 offset 적용

---

### 4. 뉴스 데이터 로딩 구조 개선
- 웹 초기 진입 시 뉴스 데이터 **1회 fetch**
- `newsStore`에서 데이터 관리
- 중복 fetch 및 데이터 불일치 문제 해결
  - 현재 list만 newsStore데이터를 가져오도록 구현
  - 추후 Grid도 동일하게 데이터를 가져오도록 통일 예정

---

### 5. 카테고리 / 페이지네이션 로직
- 카테고리 중복 제거 후 목록 생성 (`Set` 활용)
- 페이지 상태를 기준으로 현재 뉴스 및 카테고리 계산
- 페이지 이동 / 카테고리 변경 / View 전환 시 동작 기준 정리

---

## List View 렌더링 기준 정리

리스트는 **페이지 값을 기준으로 렌더링**되며 상황에 따라 다음 기준을 따름

1. **List View로 전환 시**
   - 항상 초기 페이지(첫 뉴스)부터 렌더링

2. **특정 카테고리 선택 시**
   - 해당 카테고리의 첫 뉴스부터 렌더링

3. **페이지 이동 시**
   - 현재 페이지 기준으로 다음 뉴스 렌더링

- 렌더링 과정
  - 전체 뉴스 데이터 기준으로
    - 현재 페이지가 속한 카테고리 계산
    - 화면에 표시될 뉴스 컨텐츠 계산
  - 계산 결과를 바탕으로
    - 뉴스 리스트 렌더링
    - 카테고리 active 상태 동기화
