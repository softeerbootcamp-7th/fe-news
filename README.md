  - 전체 언론사 탭 (리스트 보기)
    - 카테고리 탭 구성 (종합/경제, 방송/통신, IT, 영자지, 스포츠/연예, 매거진/전문지, 지역)
    - 선택된 카테고리의 이름 옆 해당 카테고리에 속한 언론사 개수와 현재 언론사의 순서 표시 -> 화면 새로고침 시 언론사 순서 랜덤 재배치
    - 한 언론사 당 20초간 화면 표시 -> 시간에 맞게 프로그레스 바로 표기 + 20초가 되면 다음 언론사 렌더링
    - 마지막 언론사가 20초 간 보여진 후 다음 카테고리로 이동
    - 마지막 카테고리 이후엔 처음 카테고리로 이동 (인피니트 기능)
    - 선택된 카테고리 외의 다른 카테고리에 마우스 hover 시 밑줄 표시
    - 각 카테고리 클릭 시 즉시 이동

### 프로그래밍 요구 사항
  - HTML, CSS 개발
    - 어떤 태그를 사용해야 할지 설계하고 진행한다.
    - 웹접근성에 도움이 되도록 semantic tag를 적절히 사용한다.
    - 디자인 가이드에 맞게 레이아웃과 스타일 작업을 한다.
  - 개발환경
    - Vite을 기본으로 환경구성한다.
    - vanillaJS를 기본 구성으로 한다.
    - vanilla CSS를 사용한다.
  - ESM
    - ES Modules 방식으로 개발한다. (import, export)
    - index HTML페이지에서 app.js 파일을 `type=module` 속성을 사용해서 호출한다.
    - app.js안에서는 필요한 HTML을 동적으로 렌더링하도록 한다.
    - 렌더링에 필요한 다른 JavaScript 모듈을 추가로 만들어도 좋다.

### 요구 사항 분석
  - 마우스 hover 시 롤링 정지 및 헤드라인에 밑줄 표시
    - 한쪽에 hover하면 양쪽의 헤드라인을 모두 멈추는 방법 -> 5초라는 짧은 시간과 hover한 헤드라인을 읽는다는 점을 고려했을 때 UX를 위해 양쪽을 멈춰야 놓치는 기사 없이 볼 수 있다.
    - 한쪽에 hover하면 한쪽의 헤드라인만 멈추는 방법 -> 커서가 빠져나갔을 때 다시 롤링을 다른 한쪽과 1초 차이를 맞추기가 애매해진다.

  - 다크모드 버튼
    - ON/OFF가 다크모드를 활성화하는 것인지 비활성화하는 것인지 애매하므로 해와 달 아이콘으로 주로 채택되는 우상단에 위치하도록 한다.

## 커밋 컨벤션
  - feat (feature) : 기능 구현
  - fix (bug fix) : 버그 수정
  - docs (documentation) : 문서 관련 사항
  - style (formatting, missing semi colons, …) : 스타일 변경 (형식, 세미콜론 누락 등)
  - refactor : 리팩토링
  - test (when adding missing tests) : 테스트
  - build : 빌드 관련 파일 수정
  - ci : CI 설정 파일 수정
  - perf : 성능 개선
  - chore (maintain) : 빌드, 패키지 관련 (업데이트 등) 혹은 그 외 자잘한 수정
